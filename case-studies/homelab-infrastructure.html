Infrastructure Overview

The homelab runs on a Debian 13 (Trixie) server with 16GB RAM, 6TB storage configured with LVM for flexible volume management, and Intel i5 processor. The entire infrastructure is containerized using Docker and orchestrated with Docker Compose for declarative service management.

Key services include: Prometheus for metrics collection, Grafana for visualization, Portainer for container management, Uptime Kuma for service monitoring, Watchtower for automated updates, and Fail2Ban for intrusion prevention.

Container Orchestration

All services are deployed as Docker containers with dedicated networks for isolation. Docker Compose files define service dependencies, health checks, restart policies, and resource limits. Volume mounts are used for persistent data storage with regular backup schedules. 

version: '3.8'
services:
  prometheus:
    image: prom/prometheus:latest
    volumes:
      - ./prometheus:/etc/prometheus
      - prometheus-data:/prometheus
    ports:
      - "9090:9090"
    restart: unless-stopped

  grafana:
    image: grafana/grafana:latest
    volumes:
      - grafana-data:/var/lib/grafana
    ports:
      - "3000:3000"
    restart: unless-stopped

Networking & Remote Access

Network segmentation is achieved through Docker bridge networks and UFW firewall rules. Remote access is secured via Tailscale VPN, eliminating the need for port forwarding and providing zero-trust network access. OpenWRT router handles local network routing and acts as a DNS server with custom domain resolution.

Nginx reverse proxy handles SSL/TLS termination with Let's Encrypt certificates, routing traffic to internal services based on subdomain. Cloudflare provides DNS management and DDoS protection for public-facing services.

Security Hardening

Security measures include: SSH key-only authentication with password authentication disabled, UFW firewall with default-deny policy, Fail2Ban for automated IP banning after failed login attempts, and regular security updates via unattended-upgrades.

All containers run with least-privilege principles—non-root users where possible, read-only filesystems for immutable services, and security scanning with Docker Bench for Security. Secrets are managed via Docker secrets and environment variables stored in encrypted .env files.

Monitoring Stack

Prometheus collects metrics from Node Exporter (system metrics), cAdvisor (container metrics), and service-specific exporters. Grafana dashboards provide real-time visualization of CPU, memory, disk I/O, network traffic, and container health.

Uptime Kuma monitors service availability with configurable health checks and alert notifications via Discord webhooks. Custom Prometheus alert rules trigger on critical conditions like high disk usage or service downtime.

Backup Strategy

Automated backup solution using rsync and systemd timers for scheduled incremental backups. Docker volume data is backed up to external storage daily, with weekly full system snapshots using LVM snapshots.

Configuration files and Docker Compose definitions are version-controlled in a private Git repository for disaster recovery and infrastructure-as-code practices. Backup verification scripts ensure data integrity.

Lessons Learned

  • Container Orchestration: Docker Compose simplifies multi-service management, but proper health checks and restart policies are critical for resilience.
  • Monitoring First: Implementing monitoring early in the infrastructure lifecycle enables proactive issue detection and capacity planning.
  • Security Layers: Defense-in-depth approach with multiple security layers (firewall, VPN, authentication, IDS) provides comprehensive protection.
  • Documentation: Maintaining infrastructure documentation and runbooks accelerates troubleshooting and onboarding.
  • Automation: Automated updates and backups reduce manual overhead and improve reliability.
